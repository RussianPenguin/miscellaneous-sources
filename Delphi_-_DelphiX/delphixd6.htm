DelphiX and UnDelphiX in Delphi 6

 

Introduction:

 

This tutorial will show you how to re-compile DelphiX or UnDelphiX in Delphi 6.0.  Expect the whole process to take 15 to 20 minutes, and hopefully you’ll learn something on the way.

 

Why is this a tutorial and not a patch or downloadable add-on for DelphiX?  Two reasons:

 

1)      The wording of Hori’s “TERMS AND CONDITIONS OF USE” states under no circumstances must this archive be changed. Distributing a modified archive is a violation of the software license.

2)      This is a quick fix that disables (breaks) DelphiX support for D3-D5.  I’m currently working on a {$IFDEF} version that could be submitted to Hori for inclusion into DelphiX.

 

*** DISCLAIMER - please backup your files before you do any of this, also these methods are only partially tested in Delphi 6 Personal, I haven't tested any of this on Delphi 6 Professional.

 

 

Recompiling:

 

Step 1:

 

Get the latest version of DelphiX here.

 

After Unzipping DelphiX to a directory, launch Delphi6.  Do not run Install_for?.exe or try to attempt any other installation.

 

Step 2:

 

Proceed to File...Open… DelphiX_for5.dpk and Delphi 6 will convert the package for you.

 

Resave the package as DelphiX_for6.dpk.

 

Step 3:

 

Try to click the Install button.  The compiler will catch the errors that will need to be corrected.  All of these errors reside in the DXRender.pas file.

 

[Note if you are trying to compile UnDelphiX]

 

The JEDI headers have a issue in DirectDraw.Pas. This change would be for UnDelphiX only.  

 

This code fragment…

 

var

  NilGUID : TGUID absolute 0;

 

…will need to be changed to…

 

var

  NilGUID : TGUID;

 

[Note if you are trying to compile UnDelphiX]

 

Step 4:

 

The first type of error you will need to correct is an Operand Size Mismatch error.  There are 40 to 50 errors of this type. 

 

 

 

The fix is simple...

 

mov eax,Dest; add eax,TDXRMachine_Color.R+1 // bad statement

 

…change to…

 

mov eax,Dest; add eax,BYTE(TDXRMachine_Color.R+1) // good statement

 

This simply asks for the lower 8 bits of TDXRMachine_Color.R – you will need to do this for every ASM reference to TDXRMachine_Color.  It can be accomplished with a Search and Replace.

 

Step 5:

 

The second type of error you will need to correct is an Inline Assember Syntax error.  There are 10 to 20 errors of this type, and they always occur around _AddTable and _SubTable in the source.  The fix is just as simple...

 

movzx edx,byte [offset _AddTable+edx+ebx]

 

…change to…

 

movzx edx,byte [offset (_AddTable+edx+ebx)]

 

After all these errors have been addressed compiling the package will show only one warning and one fatal error.

 

Step 6:

 

Lastly, and probably the most confusing part of this Delphi’s re-nameing of DsgnIntf in the new versions of Delphi.  Because of this DXReg.pas will not compile. 

 

 

There are two ways around the issue…

 

No IDE editors:

 

First delete everything under the Type declaration in DXReg.Pas, then remove all references to RegisterComponentEditor and RegisterPropertyEditor in the Register procedure.  Finally remove all code below the RegisterComponents  procedure, making sure DXReg.pas still ends with an ‘end.’

 

When you are done with all that hacking your DXReg file should look like this…

 

unit DXReg;

 

interface

 

uses

  Windows, SysUtils, Classes, Forms, Dialogs, Graphics, TypInfo,

  DXDraws, DXSounds, DIB, Wave, DXInput, DXPlay, DXSprite,

  DXClass;

 

procedure Register;

 

implementation

 

const

  SNone = '(None)';

 

  SSettingImage = '&Image...';

  SSettingWave = '&Wave...';

  SDXGFileFilter = 'DXG file(*.dxg)|*.dxg|All files(*.*)|*.*';

  SDXGOpenFileFilter = 'DXG file(*.dxg)|*.dxg|Bitmap file(*.bmp)|*.bmp|All files(*.*)|*.*';

  SDXWFileFilter = 'DXW file(*.dxw)|*.dxg|All files(*.*)|*.*';

  SDXWOpenFileFilter = 'DXW file(*.dxw)|*.dxw|Wave file(*.wav)|*.wav|All files(*.*)|*.*';

  SSinglePlayer = '&Single player';

  SMultiPlayer1 = 'Multi player &1';

  SMultiPlayer2 = 'Multi player &2';

 

  SOpen = '&Open...';

  SSave = '&Save..';

 

procedure Register;

begin

  RegisterComponents('DelphiX',

    [TDXDraw,

    TDXDIB,

    TDXImageList,

    TDX3D,

    TDXSound,

    TDXWave,

    TDXWaveList,

    TDXInput,

    TDXPlay,

    TDXSpriteEngine,

    TDXTimer,

    TDXPaintBox]);

end;

 

end.

 

 

With this solution you will not be able use any fancy property editors that are normally included in DelphiX.

 

With IDE editors (experts):

 

A known workaround for the missing 'DsgnIntf' for professional versions of Delphi was recommended to me by Tommy "QuazzeMan" Jonsson and posted on the ExpertExchange.

1) In the unit DXReg.pas where it USES 'DsgnIntf' replace that with 'DesignIntf'.
2) add 'DesignEditors' to the uses clause of the unit and then also
3) add 'DesignIDE' to the REQUIRED section of the package

It has been verified to work in all versions of Delphi 6.

 

Step 7:
